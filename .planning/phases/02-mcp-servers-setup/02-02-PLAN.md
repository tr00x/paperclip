Using the Telegram credentials from Task 1:

1. Deploy Mailpit via Docker:
   ```
   mkdir -p mcp-servers/mailpit
   cat > mcp-servers/mailpit/docker-compose.yml << 'EOF'
   name: mailpit
   services:
     mailpit:
       image: axllent/mailpit:latest
       restart: unless-stopped
       ports:
         - "8025:8025"   # Web UI
         - "1025:1025"   # SMTP
         - "1143:1143"   # IMAP
       environment:
         MP_SMTP_AUTH_ACCEPT_ANY: 1
         MP_SMTP_AUTH_ALLOW_INSECURE: 1
   EOF
   cd mcp-servers/mailpit && docker compose up -d
   ```

2. Verify Mailpit is running:
   ```
   curl -s http://localhost:8025/api/v1/info | grep "Mailpit"
   ```
   Expected: JSON with Mailpit version info.

3. Create `mcp-servers/telegram/.env`:
   ```
   TELEGRAM_BOT_TOKEN=<user-provided token>
   TELEGRAM_CHAT_ID=<user-provided chat_id>
   ```

4. Register Telegram MCP in Claude Code project scope:
   ```
   claude mcp add telegram \
     -e TELEGRAM_BOT_TOKEN=<token> \
     -e TELEGRAM_CHAT_ID=<chat_id> \
     -- npx @parthj/telegram-notify-mcp
   ```

5. Verify Telegram bot by sending a test message via curl:
   ```
   curl -s -X POST "https://api.telegram.org/bot<TOKEN>/sendMessage" \
     -d "chat_id=<CHAT_ID>&text=AmriTech AI HQ connected! MCP setup complete."
   ```
   Expected: JSON response with `"ok":true`.

6. Create `mcp-servers/email/.env`:
   ```
   MCP_EMAIL_ADDRESS=agents@amritech.local
   MCP_EMAIL_PASSWORD=anything
   MCP_EMAIL_IMAP_HOST=localhost
   MCP_EMAIL_IMAP_PORT=1143
   MCP_EMAIL_SMTP_HOST=localhost
   MCP_EMAIL_SMTP_PORT=1025
   ```

7. Register Email MCP in Claude Code project scope:
   ```
   claude mcp add email \
     -e MCP_EMAIL_ADDRESS=agents@amritech.local \
     -e MCP_EMAIL_PASSWORD=anything \
     -e MCP_EMAIL_IMAP_HOST=localhost \
     -e MCP_EMAIL_IMAP_PORT=1143 \
     -e MCP_EMAIL_SMTP_HOST=localhost \
     -e MCP_EMAIL_SMTP_PORT=1025 \
     -- npx @codefuturist/email-mcp stdio
   ```

8. Verify Email MCP by sending a test email through Mailpit SMTP:
   ```
   # Send test email via SMTP to Mailpit
   curl -s --url "smtp://localhost:1025" \
     --mail-from "agents@amritech.local" \
     --mail-rcpt "test@example.com" \
     --upload-file - << 'MAIL'
   From: AmriTech AI <agents@amritech.local>
   To: test@example.com
   Subject: MCP Email Test

   This is a test email from AmriTech AI HQ.
   If you see this in Mailpit, email MCP is working.
   MAIL
   ```
   Then verify it appeared in Mailpit:
   ```
   curl -s http://localhost:8025/api/v1/messages | grep -q "MCP Email Test" && echo "EMAIL_OK"
   ```
   Expected: "EMAIL_OK" — email caught by Mailpit.

9. Update `mcp-servers/README.md`:
   - Mark telegram as "Configured"
   - Mark email as "Configured (Mailpit local)"
   - All 5 servers should now show "Configured"
   - Add Mailpit section: `Web UI: http://localhost:8025`, `SMTP: localhost:1025`, `IMAP: localhost:1143`
   - Note: "To switch to real email, change SMTP_HOST/PORT in mcp-servers/email/.env to your provider (SendGrid, Mailgun, etc.)"

10. Ensure `mcp-servers/*/.env` lines are in `.gitignore`.
