# ЧАСТЬ 1: БЫСТРЫЙ СТАРТ (5 минут)

## Что такое AI-штаб?

У нас работают **12 AI-сотрудников** — они ищут клиентов, пишут email'ы, следят за контрактами и деньгами. Они работают 24/7, не спят, не болеют, не забывают.

**Ты + Berik + Ula + 12 AI-агентов = команда из 15 человек.**

## Что тебе нужно делать?

Всего 3 вещи:

1. **Telegram** — общайся с агентами через команды
2. **CRM** — записывай результаты звонков и встреч
3. **Реагируй** — когда агент просит тебя что-то сделать — делай

## Твои первые 5 минут

1. Открой **Telegram** → наш групповой чат
2. Напиши `/help` → прочитай список команд
3. Напиши `/status` → посмотри кто из агентов работает
4. Напиши `/pipeline` → посмотри сколько лидов в работе
5. Готово! Ты в системе.

# ЧАСТЬ 2: TELEGRAM — КАК ОБЩАТЬСЯ С АГЕНТАМИ

## Где это?

В нашем **групповом чате** в Telegram. Все команды пишутся прямо туда.

## Быстрые команды (ответ за 1 секунду)

Эти команды не будят агентов — бот сам отвечает мгновенно:

| Что написать | Что увидишь |
|-------------|------------|
| `/status` | Список агентов: 🟢 работает, 🟡 задержка, 🔴 проблема |
| `/pipeline` | Сколько лидов на каждом этапе (new, contacted, replied...) |
| `/leads` | Сколько email'ов отправлено, кто ответил |
| `/help` | Полная справка со всеми командами |

**Пример:** Пишешь `/status`, получаешь:
```
📊 Статус штаба

Агенты:
🟢 CEO — 2ч назад
🟢 Hunter — 1ч назад
🟢 SDR — 30м назад
🟡 Closer — idle (нет задач)
🔴 Finance Tracker — 3 дня назад

Сервисы: CRM: 🟢 | Sync: 🟢
```

## Команды агентам (создают задачу и будят агента)

| Что написать | Кто получит | Пример |
|-------------|-----------|--------|
| `/hunter найди...` | Hunter (искатель) | `/hunter найди стоматологии в Bergen County NJ` |
| `/sdr напиши...` | SDR (email) | `/sdr напиши email для ABC Dental` |
| `/closer подготовь...` | Closer (briefing) | `/closer подготовь brief для звонка с ABC Corp` |
| `/contract проверь...` | Contract Manager | `/contract проверь renewal для ABC Corp` |
| `/finance сколько...` | Finance Tracker | `/finance сколько MRR сейчас?` |
| `/onboard запусти...` | Onboarding | `/onboard запусти онбординг для XYZ Law` |
| `/staff что...` | Staff Manager | `/staff что умеет Hunter?` |
| `/fix проблема` | IT Chef (починка) | `/fix CRM не грузится` |
| `/gov проверь...` | Gov Scout (тендеры) | `/gov проверь тендеры на IT в NJ` |
| `/proposal напиши...` | Proposal Writer | `/proposal напиши КП для ABC Corp` |

**Просто текст без `/`** → получит CEO (координатор):
```
добавь эту компанию в приоритет: ABC Inc
```

## Примеры (копируй и вставляй)

```

**Хочу узнать что происходит:**
```

## Как отправить фото или файл агенту

1. Нажми скрепку 📎 в Telegram
2. Выбери фото или файл
3. В подписи (caption) напиши команду

**Примеры:**
- Фото визитки + подпись `/hunter новый контакт` → Hunter разберёт визитку
- Скриншот ошибки + подпись `/fix` → IT Chef диагностирует
- PDF контракт + подпись `/legal проверь` → Legal ревьюит

## Что делать когда агент пишет ТЕБЕ

Агенты будут писать в чат и тегать тебя (@ikberik или @UlaAmri). Это значит им нужно от тебя действие.

### Пример 1: SDR просит одобрить ответ
```
📧 SDR — Получен ответ!
От: John Smith (ABC Corp)
Тема: Re: On-site IT for your NJ office
Суть: Interested in meeting next week
Тон: позитивный
Жду указаний — отвечать?
```

**Что делать Berik:** Ответь в чат: `Да, ответь ему. Предложи вторник или среду 10 AM.`

### Пример 2: Contract Manager просит Ula позвонить
```
⚠️ @UlaAmri, контракт ABC Corp истекает через 15 дней.
SDR outreach без ответа. Позвони клиенту!
```

**Что делать Ula:** Позвони клиенту. После звонка запиши результат в CRM (см. Часть 3).

### Пример 3: Finance просит решение
```
💰 @ikberik, счёт XYZ Corp просрочен 14 дней ($3,000). Решение?
```

**Что делать Berik:** Ответь: `Подожди ещё неделю` или `Ula, позвони им` или `Отправь formal reminder`.

### Пример 4: SDR спрашивает про отправку
```
📧 Готово 3 письма. Отправить сейчас или запланировать на Пн 9AM?
```

**Что делать Berik:** Ответь: `Отправляй` или `На понедельник`.

## Если агент пингует повторно

Агенты **требовательные**. Если ты не ответил — они напомнят:

- Через 2 часа: "Лид ответил 2ч назад. @ikberik, решение?"
- Через 4 часа: "⚠️ Лид остывает!"
- Через 8 часов: "🔴 СРОЧНО! Лид ждёт 8 часов!"

**Это нормально.** Они делают это потому что каждый час промедления = клиент может уйти к конкуренту. Просто ответь когда увидишь.

# ЧАСТЬ 3: CRM — КАК РАБОТАТЬ С БАЗОЙ ДАННЫХ

## Что такое CRM?

CRM (Twenty) — это наша база данных. Здесь хранится информация о лидах (потенциальных клиентах), текущих клиентах, контрактах, оплатах.

**Все AI-агенты читают данные из CRM.** Если ты не записал результат звонка — агенты не знают что произошло.

## Как открыть CRM

**Сейчас:** Только с компа Tim'а → открой браузер → `localhost:5555`
**Скоро:** Из любого браузера → `crm.amritech.us` (настроим завтра)

## Как войти

1. Открой CRM в браузере
2. Логин: (спроси у Tim'а или Berik'а — он настраивал)
3. Увидишь dashboard с агентами, задачами, лидами

## Что ты видишь в CRM

### Левое меню:
- **Leads** — потенциальные клиенты (от Hunter)
- **Clients** — действующие клиенты
- **Invoices** — счета
- **People** — контактные лица
- **Companies** — компании

### Карточка лида:

Когда открываешь лид, видишь:

| Поле | Что это | Пример |
|------|---------|--------|
| Name | Название компании | ABC Dental Practice |
| Decision Maker | Контактное лицо | John Smith |
| DM Email | Его email | john@abcdental.com |
| DM Phone | Его телефон | (718) 555-1234 |
| ICP Score | Оценка качества лида (0-100) | 72 |
| Industry | Ниша | Dental |
| Location | Где находятся | Brooklyn, NY |
| Status | Где в воронке | contacted |
| Outreach Status | Что происходит с email'ами | email_sent |
| Notes | Заметки (сюда ТВОИ записи!) | — |

## Статусы лида (что они значат)

| Что видишь | Что это значит | Что происходит |
|-----------|---------------|---------------|
| **new** | Hunter только нашёл | Ждём пока SDR напишет email |
| **qualified** | Готов к outreach | SDR берёт в работу |
| **contacted** | Email отправлен | Ждём ответ |
| **engaged** | Лид ответил! | Berik решает что делать |
| **meeting_set** | Звонок назначен | Closer готовит briefing |
| **closed_won** | Стал клиентом! 🎉 | Onboarding запускается |
| **closed_lost** | Отказал | Архив |
| **nurture** | Не сейчас | Вернёмся через 3 месяца |

**Ты НЕ меняешь статусы вручную** — агенты делают это автоматически. Твоя задача — писать заметки (Notes).

# ЧАСТЬ 4: КТО ЕСТЬ КТО — АГЕНТЫ

## Список всех агентов

### 🔍 Hunter — Искатель лидов
**Что делает:** Ищет компании которым нужен IT. Проверяет Google Maps, LinkedIn, Indeed, отзывы. Оценивает по ICP score (0-100).
**Работает:** 4 раза в день (каждые 6 часов)
**Результат:** Новый лид в CRM с оценкой и контактами
**Пример:** "Нашёл стоматологию в NJ с истёкшим SSL сертификатом — ICP 72/100"

### 📧 SDR — Писатель email'ов
**Что делает:** Пишет персонализированные email'ы лидам. Отправляет, делает follow-up на Day 3 и Day 7. Проверяет ответы.
**Работает:** Каждые 2 часа
**Результат:** Email отправлен, ответ обработан
**Важно:** Спрашивает у Berik'а подтверждение. Отправляет только Пн-Чт 8-10 AM.

### 🤝 Closer — Подготовка к звонку
**Что делает:** Когда лид ответил положительно — готовит полный briefing для Berik'а: кто клиент, что болит, какая цена, какие возражения ожидать, как вести разговор.
**Работает:** По задаче (когда лид ответил)
**Результат:** Berik идёт на звонок полностью подготовленный

### 👑 CEO — Координатор штаба
**Что делает:** Раздаёт задачи всем агентам, делает утренний дайджест, еженедельные отчёты, следит за pipeline.
**Работает:** Утром (9 AM) + вечером (6 PM) + по событиям
**Результат:** Утренний дайджест в TG, weekly report

### 🛟 Staff Manager — Надзиратель
**Что делает:** Следит за здоровьем агентов. Если кто-то завис — сигнализирует. Следит чтобы Berik и Ula вели CRM.
**Работает:** Каждые 4 часа
**Пример:** "SDR не отправил ни одного email за 24 часа — что-то не так!"

### 📋 Contract Manager — Контракты
**Что делает:** Следит за сроками контрактов. За 30 дней до окончания — просит Ula позвонить. Если молчат — эскалирует.
**Работает:** Ежедневно
**Пример:** "Контракт ABC Corp через 30 дней — @UlaAmri, позвони"

### 💰 Finance Tracker — Финансы
**Что делает:** Считает MRR (ежемесячный доход), следит за оплатами, эскалирует просрочки.
**Работает:** По понедельникам
**Пример:** "Счёт XYZ просрочен 14 дней — @ikberik, решение?"

### 🔧 IT Chef — Девопс
**Что делает:** Мониторит все сервисы (CRM, Paperclip, email, Telegram). Если что-то упало — чинит автоматически. Когда Tim offline — его полная замена.
**Работает:** Каждый час
**Пример:** "CRM sync упал — перезапустил, всё работает"

### 📝 Proposal Writer — Proposals
**Что делает:** Пишет коммерческие предложения (КП), ответы на тендеры (RFP), capability statements.
**Работает:** По задаче
**Результат:** Готовый документ (.docx) для клиента

### ⚖️ Legal Assistant — Юрист
**Что делает:** Проверяет контракты на риски, compliance (HIPAA, SOX), находит проблемные пункты.
**Работает:** По задаче
**Пример:** "В контракте нет SLA — нужно добавить перед подписью"

### 🚀 Onboarding Agent — Онбординг
**Что делает:** Когда новый клиент подписал — отправляет welcome email, IT аудит чеклист, 30-дневный план.
**Работает:** По задаче
**Пример:** "Отправил welcome пакет новому клиенту"

### 🏛️ Gov Scout — Тендеры
**Что делает:** Мониторит SAM.gov, NY/NJ порталы. Ищет IT тендеры, скорит по размеру и сложности.
**Работает:** Ежедневно
**Пример:** "Тендер на IT services в Newark — score 78/100, deadline через 14 дней"

# ЧАСТЬ 5: КАК РАБОТАЕТ ВОРОНКА ПРОДАЖ

## Путь лида от находки до клиента

```
1. Hunter находит компанию с плохим IT
         ↓
2. Создаёт лид в CRM (ICP score, контакты, сигналы)
         ↓
3. Если ICP 60+ и есть email → SDR автоматом получает задачу
         ↓
4. SDR пишет персонализированный email
         ↓
5. Berik подтверждает время отправки
         ↓
6. SDR отправляет (Пн-Чт, 8-10 AM)
         ↓
7. Через 3 дня → автоматический follow-up #1
         ↓
8. Через 7 дней → автоматический follow-up #2
         ↓
9. Лид ответил?
   │
   ├── ДА (хочет поговорить)
   │   → SDR уведомляет в TG
   │   → Berik одобряет ответ
   │   → Closer готовит briefing
   │   → Berik звонит
   │   → Если закрыли → Onboarding + Contract + Finance автоматом
   │
   ├── ДА (вопрос по цене)
   │   → SDR готовит ответ → Berik одобряет → SDR отправляет
   │
   ├── ДА (не интересно)
   │   → SDR закрывает вежливо → лид в архив
   │
   └── НЕТ (тишина)
       → Через 14 дней → финальное письмо → лид в nurture (вернёмся через 90 дней)
```
