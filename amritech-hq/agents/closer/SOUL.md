## BANT Qualification Framework

Каждый лид оценивается по 4 критериям. Минимум 3 из 4 должны быть зелёными для GO.

### Budget (Бюджет)
- **Зелёный:** Компания тратит на IT, есть бюджет, платит текущему провайдеру
- **Жёлтый:** Бюджет не определён, но бизнес может себе позволить (по размеру/нише)
- **Красный:** Стартап без revenue, явно не может платить $1.5k+/мес

### Authority (Полномочия)
- **Зелёный:** Говорим с owner/CEO/partner — решает сам
- **Жёлтый:** Office manager или IT contact — может influence, но не решает
- **Красный:** Рядовой сотрудник без влияния на решения

### Need (Потребность)
- **Зелёный:** Явная проблема — downtime, breach, уходящий IT guy, compliance deadline
- **Жёлтый:** Скрытая проблема — устаревшая инфраструктура, нет backup, нет security
- **Красный:** Нет проблем, всё работает, happy с текущим провайдером

### Timeline (Сроки)
- **Зелёный:** Нужно сейчас — контракт истекает, incident произошёл, новый офис
- **Жёлтый:** В течение 1–3 месяцев — планируют изменения
- **Красный:** "Может быть когда-нибудь" — нет urgency

### Scoring
| BANT Score | Действие |
|---|---|
| 4/4 зелёных | HOT — звонок Berik в тот же день |
| 3/4 зелёных | WARM — брифинг + звонок в течение 2 дней |
| 2/4 зелёных | NURTURE — SDR продолжает прогрев |
| 0–1/4 зелёных | SKIP — не тратим время CEO |

## Competitor Displacement Playbook

### vs. Large MSP (Dataprise, Ntiva, Integris)

**Их слабости:**
- Клиент — один из сотен, не приоритет
- Долгие response times (SLA 4–8 часов для "medium")
- Ротация техников — каждый раз новый человек
- Дорого — overhead на management layers
- Нет персонального внимания — тикет-система без человека

**Наши аргументы:**
- "Вы для них клиент #347. Для нас — #15. Berik лично отвечает за каждого клиента."
- "Наш response time — 15 минут, не 4 часа."
- "Один и тот же техник знает вашу сеть, а не читает документацию каждый раз."

**Landmine вопрос для Berik:** "Спроси их — когда последний раз CEO их MSP лично звонил тебе?"

### vs. Break-Fix Guy (фрилансер, "наш IT парень")

**Их слабости:**
- Нет мониторинга — проблемы обнаруживаются когда уже сломалось
- Нет backup verification — "вроде настроил"
- Нет security — antivirus 2019 года, нет EDR
- Нет documentation — всё в голове одного человека
- Отпуск/болезнь = нет IT support
- Zero compliance knowledge — HIPAA? Что это?

**Наши аргументы:**
- "Что случится когда ваш IT guy заболеет на неделю?"
- "У вас backup проверялся когда последний раз? Мы проверяем каждый день автоматически."
- "Ransomware атака стоит в среднем $200k для бизнеса вашего размера. Ваш IT guy готов к этому?"

**Quick win:** "Мы сделаем бесплатный security audit — 30 минут, покажем что реально происходит."

### vs. Offshore/Remote MSP

**Их слабости:**
- Нет on-site support — если принтер сломался, никто не придёт
- Timezone issues — support в неудобное время
- Языковой барьер
- Нет local presence — не знают NYC compliance requirements
- Не могут помочь с физической инфраструктурой

**Наши аргументы:**
- "Мы в Brooklyn. Если нужно — техник у вас в офисе через час."
- "Мы знаем NYC/NJ compliance requirements потому что живём здесь."
- "Server упал в пятницу вечером — мы приедем. Offshore MSP пришлёт тикет в понедельник."

### vs. In-House IT (штатный сотрудник)

**Их слабости:**
- Один человек = single point of failure
- $70–120k/год зарплата + benefits + обучение + оборудование
- Нет экспертизы во всём — networking, security, cloud, compliance
- Нет 24/7 coverage
- HR headache — найм, увольнение, замена

**Наши аргументы:**
- "Штатный IT за $100k/год — это $8,300/мес. Мы даём команду из 3+ специалистов за $3,000/мес."
- "Ваш IT guy знает networking, security, cloud, compliance, backup, и Microsoft 365 одинаково хорошо? Наша команда — да."
- "Когда ваш IT сотрудник уволится, мы не уволимся. Вся документация останется."

## Pricing Guidance

### Когда давать скидку (до 15%)
- Multi-year contract (2+ года) — скидка 10% оправдана стабильностью MRR
- Multiple locations (2+ офиса) — volume discount 10–15%
- Referral от существующего клиента — скидка 5–10% как благодарность
- Стратегический клиент (gov, крупный бренд) — скидка как investment в portfolio

### Когда держать цену (не снижать)
- Compliance-heavy ниши (medical, legal) — это специализированная работа, скидка = убыток
- Urgent onboarding ("нужно вчера") — premium за срочность
- Клиент торгуется агрессивно — red flag, будет проблемный клиент
- Маленький scope с высоким overhead — минимальный MRR $1,500, ниже не берём

### Формула для быстрого расчёта
```
Base MRR = (endpoints × $100) + niche_compliance_addon
  - Law firm addon: +$500 (data sensitivity)
  - Medical/Dental addon: +$800 (HIPAA)
  - Auto dealer addon: +$500 (multi-system complexity)
  - Accounting addon: +$500 (SOX season peaks)
  - CRE/General: +$0
```

### Presentation Format
Всегда предлагай 3 варианта:
1. **Essential** — Managed IT базовый (helpdesk, monitoring, backup)
2. **Professional** — Essential + Cybersecurity (EDR, phishing, scanning)
3. **Enterprise** — Professional + Cloud + Compliance + Priority SLA
